TFidf vs NL Embeddings for Text Classification¶

Objectives¶

In this notebook, I will compare a classical machine learning baseline text classification using TF-IDF with logistic regression of sklearn, against neural model using embeddings with linear classifier.
The performance of the two models shall be evaluated by precision, recall and f1-score metrics, and then predictions visualised by confusion matrix.

A subsample of the tweet dataset will be used for the modelling. The tweet dataset is already cleaned from the previous data preparations with pyspark, therefore the cleaned data shall be loaded.

Result¶

The performance of the baseline model was better than the neural network models. Reflecting on this performance, the first neural network was overfitting the data, while the second neural network reached convergence after approximately six epochs and further trainings yielded no improvement in the model. The model with LSTM had improved validation loss but plateaued after about 10 epochs. These are evidenced by the training vs validation loss plots shown in the epoch vs loss plots. This poor performance of the neural models could be attributed to the fact that the neural model was trained on a very small set of data and even after the addition of regularisation, the data is not large enough for model learning. Neural models are best trained on a larger dataset with good regularization which enables the model to learn the relationship in the data and be able to generalize better on unseen data. The first neural model is only a single perceptron which had no regularization leading to low inductive bias with low data. NLP require deeper networks and transformers to enable to model learn the semantics between words and be able to generalize better. On the other hand, logistic regression in classical machine learning generalize better for sparse and small to medium dataset (ca 100k). Also TF-IDF encodes term importance explicitly while ignoring semantics between words. However, embeddings are learning semantics which therefore needs dense and larger dataset in order to build stronger models.

Overall, the training and prediction time for logistic regression was unsurprisingly low given the number of parameters compared to the neural models. One thing to note on the neural model is that the improved neural model used a smaller/modified architecture alongside regularisation, which reduced both parameter count and training time. Although regularisation and LSTM improved training dynamics, the predictive performance remained close to chance level on this balanced dataset

If this model were to be deployed, I would deploy the classical baseline logistic regression model because this model was able to generalize more on the trained dataset. In the future if the dataset were to increase and the neural network is trained with transformers and performs better on a larger dataset, then that could be deployed.

Next steps¶

What I would try next to improve the model would be:

pretrained embeddings
larger training sample
bidrectional LSTM
transformer encoder and
error analysis on misclassified tweets.

def tokenize(text):
    return text.lower().split()

def build_vocab(dataset, max_size=None):
    counter = Counter()
    for text in dataset:
        counter.update(tokenize(text))

    vocab = {
    "<unk>": 0, # setting any word that is not in the vocabulary to zero
    "<pad>":1  # setting the padding index to 1
    }
    for word, _ in counter.most_common(max_size):
        if word not in vocab:
            vocab[word] = len(vocab)

    return vocab

In the vocabulary dictionary, words not in the dictionary are returned as zero and the number 1 is set to padding. This is because neural models require samples sequence to be of same length within a training batch.

class CustomTextData(Dataset):
    def __init__(self, texts, labels, vocabulary, numericalize):
        self.texts = texts
        self.labels = labels 
        self.vocabulary = vocabulary
        self.numericalize = numericalize

    def __getitem__(self, index):
        text = self.texts[index]
        label = self.labels[index]
        numericalized_word = self.numericalize(text)
        return numericalized_word, label

    def __len__(self):
        return len(self.labels)

def collate_batch(batch):
    label_list, text_list = [], []
    for (_text, _label) in batch: 
        label_list.append(_label)
        processed_text = torch.tensor(_text)
        text_list.append(processed_text)

    padded_text = pad_sequence(text_list, batch_first=True, padding_value=1.0)
    return padded_text.to(device), torch.tensor(label_list, dtype=torch.float64).to(device)

class TextClassifier(nn.Module):
    def __init__(self, vocabulary_size, embedding_dim, output_dim):
        super().__init__()
        self.embedded = nn.Embedding(vocabulary_size, embedding_dim)
        self.average_pooling = nn.AdaptiveAvgPool1d(1)
        self.linear_stack = nn.Sequential(
                                    nn.Linear(embedding_dim, output_dim),
        )

    def forward(self, x):
        embedding = self.embedded(x)
        embedding = embedding.permute(0, 2, 1)
        pooled_avg = self.average_pooling(embedding).squeeze(-1)
        logits = self.linear_stack(pooled_avg)
        return logits

class EarlyStopping: 
    def __init__(self, patience=5, delta=0.01, verbose=True, path="checkpoint.pt"):
        self.patience = patience
        self.verbose = verbose
        self.delta = delta
        self.path = path
        self.wait_count = 0
        self.best_score = None
        self.dynamic_patience = self.patience

    def check_early_stop(self, val_loss, model): 
        if self.best_score is None or val_loss < self.best_score - self.delta:
            self.best_score = val_loss
            self.wait_count = 0 
            self.dynamic_patience = self.patience
            torch.save(model.state_dict(), self.path) # save checkpoint
        else:
            self.wait_count += 1
            #adjust patience if improvement is near
            if self.wait_count >= (self.patience * 0.8):
                self.dynamic_patience += 1
            if self.wait_count >= self.dynamic_patience:
                if self.verbose:
                    print("Stopping early due to lack of improvement")
                return True

        return False

early_stopping = EarlyStopping(patience=3, delta=0.001)

An adaptice early stopping is defined to be used in the model training in order to stop the training there is no further improvement in the validation loss after a set number of training cycles. This will help to save compute power.
Also, for each complete pass in the data, the best validated model is saved ahead of the early stopping such that if the training is terminated, the best trained model is saved for reuse without the need to retrain the model.

def train(dataloader, model, loss_fn, optimizer):
    size=len(dataloader.dataset)
    num_batches = len(dataloader)

    model.train()
    train_loss, train_acc = 0, 0
    for batch, (X, y) in enumerate(dataloader):
        X, y = X.to(device), y.to(device)
        optimizer.zero_grad()
        predictions = model(X).view(-1)
        loss = loss_fn(predictions, y)
        round_pred = torch.round(torch.sigmoid(predictions))
        correct = (round_pred == y).type(torch.float).sum().item()
        train_acc += correct
        train_loss += loss

        loss.backward()
        optimizer.step()
        

        if batch % 500 == 0:
            loss, current = loss.item(), (batch+1)* len(X)
            print(f"Loss: {loss:.5g}  [current: {current:>5d}/{size:>5d}]")
    train_loss /= num_batches
    train_acc /= size

    print(f"Train Error: \n Accuracy: {(100*train_acc):>0.1f}%, Avg loss: {train_loss:>8f} \n")
    return train_loss, train_acc

def test(dataloader, model, loss_fn):
    size = len(dataloader.dataset)
    num_batches = len(dataloader)


    model.eval()
    test_loss, accuracy = 0, 0
    with torch.no_grad():
        for X, y in dataloader:
            X, y = X.to(device), y.to(device)
            pred = model(X).view(-1)
            test_loss += loss_fn(pred, y).item()
            round_pred = torch.round(torch.sigmoid(pred))
            correct = (round_pred == y).type(torch.float).sum().item()
            
            accuracy += correct
    
    test_loss /= num_batches
    accuracy /= size

    print(f"Test error: \n Accuracy: {(100*accuracy):>0.1f}%, Avg loss: {test_loss:>8f} \n")
    return test_loss, accuracy

class nnClassifier(nn.Module):
    def __init__(self, vocabulary_size, embedding_dim, output_dim):
        super().__init__()
        self.embedded = nn.Embedding(num_embeddings=vocabulary_size, 
                                     embedding_dim=embedding_dim,
                                     padding_idx = 1) # adding the index of padding to avoid the model learning the parameter
        self.linear_relu_stack = nn.Sequential(
                                    nn.Dropout(0.3),
                                    nn.Linear(embedding_dim, 100),
                                    nn.ReLU(),
                                    nn.Linear(100, output_dim)
        )

    def forward(self, x):
        embedding = self.embedded(x)
        pooled_avg = embedding.mean(dim=1)
        logits = self.linear_relu_stack(pooled_avg)
        return logits

class LSTMClassifier(nn.Module):
    def __init__(self, vocabulary_size, embedding_dim, hidden_size, output_dim):
        super().__init__()
        self.embedded = nn.Embedding(num_embeddings=vocabulary_size, 
                                     embedding_dim=embedding_dim,
                                     padding_idx = 1)
        self.lstm = nn.LSTM(input_size=embedding_dim,
                            hidden_size=hidden_size,
                            num_layers=1, 
                            batch_first=True
                           )
        self.linear = nn.Linear(hidden_size, output_dim)

    def forward(self, x):
        embedding = self.embedded(x)
        lstm_output, (h_n, c_n) = self.lstm(embedding)
        last_hidden = h_n[-1, :, :]
        logits = self.linear(last_hidden)

        return logits

Summary of Results¶

The performance of the baseline model was better than the neural network models. Reflecting on this performance, the first neural network was overfitting the data, while the second neural network reached convergence after approximately six epochs and further trainings yielded no improvement in the model. The model with LSTM had improved validation loss but plateaued after about 10 epochs. These are evidenced by the training vs validation loss plots shown in the epoch vs loss plots. This poor performance of the neural models could be attributed to the fact that the neural model was trained on a very small set of data and even after the addition of regularisation, the data is not large enough for model learning. Neural models are best trained on a larger dataset with good regularization which enables the model to learn the relationship in the data and be able to generalize better on unseen data. The first neural model is only a single perceptron which had no regularization leading to low inductive bias with low data. NLP require deeper networks and transformers to enable to model learn the semantics between words and be able to generalize better. On the other hand, logistic regression in classical machine learning generalize better for sparse and small to medium dataset (ca 100k). Also TF-IDF encodes term importance explicitly while ignoring semantics between words. However, embeddings are learning semantics which therefore needs dense and larger dataset in order to build stronger models.

Overall, the training and prediction time for logistic regression was unsurprisingly low given the number of parameters compared to the neural models. One thing to note on the neural model is that the improved neural model used a smaller/modified architecture alongside regularisation, which reduced both parameter count and training time. Although regularisation and LSTM improved training dynamics, the predictive performance remained close to chance level on this balanced dataset

If this model were to be deployed, I would deploy the classical baseline logistic regression model because this model was able to generalize more on the trained dataset. In the future if the dataset were to increase and the neural network is trained with transformers and performs better on a larger dataset, then that could be deployed.
